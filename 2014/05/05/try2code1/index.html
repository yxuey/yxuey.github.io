try2code计划启动！

By YuXueying

5月 5 2014 更新日期:5月 5 2014

文章目录

  2. 2. 二、把代码托管到GitHub上

沉寂好久，终于又开始写博客了。最初是打算把博客弄成偏理性的空间，还想在博客里写一些看起来高大上的笔记。但很不幸，我修行不够，所以只能再发一篇水文。

陆陆续续地看了一些前端开发方面的书籍，一直琢磨着该做点什么像样的东西，但因为各种原因迟迟没有下手。直到前段时间在微博上看到一条微博介绍了一个180天制作180个网站的艺术专业女生，当时就点进去看她的作品了。她从开始做第一个网页到后面的第180个网页，再到后来开发的YumHacker项目，一步一步学过来，进步速度之快简直令人赞叹!

我也想尝试“做网页+写博客记录”这种学习模式，因此定下了“try2code”计划，并有了我的第一个网页 （PS：电脑访问时排版很正常，移动终端访问则会出现很难看的效果，我也不知该怎么解决 T_T）。

首先，我做了一下的准备工作：

  • 做一个主页用来堆放以后的作品
  • 把代码托管到GitHub上面
  • 在GitHub上为try2code创建项目主页

下面将分享一下在每一步中我遇到的困难及学习体会：

一、制作主页

“只是需要做一个堆放作品的页面而已，应该不难的。”起初我是这么想的。后来才发觉：自己虽然看了那么多教材也跟着做了些章末练习，但要自己亲手做出一个好看的页面来还真不容易。既然是主页，肯定要简洁又不能太难看。一开始自己做了几个页面，都觉得不满意。于是寻求模仿，一开始想模仿某个漫画家的网站主页,后来因太难转而模仿另一个排版相似的页面。

这次模仿让我对CSS中的继承、选择器等有更加直观的认识，也学会了在Chrome中利用审查元素来调整页面。至于JavaScript,我仍然限于勉强能看懂的状态。明明看了一大半本教程，要自己写js还是觉得很费力，理论跟实践果然是两码事。

二、把代码托管到GitHub上

由于在github上我完全是新手，当然要找一些入门指导。首先我看了一篇博文如何高效利用GitHub,有了大致的认识。接着看了github的官方书籍《pro-git》和蒋鑫写的《Git权威指南》，还搜到了Git常用命令速查表。一知半解的情况下，我开始尝试提交。提交指令其实就那么简单的几句：

$ git status  #查看文件状态
$ git add <file-name> #跟踪指定的文件
$ git commit  #提交所有更新且跟踪过的文件
$ git push <remote-name> <branch-name> #推送数据到远程仓库

但我折腾了好久，会了之后回头去看才知道自己走了多少弯路。

刚接触一个全新的事物时，你可以通过搜集资料来获得对它的认知，然后就会觉得很简单啊就那么回事啊，一尝试你就知道错了。因为会出现各种你难以理解的错误，而且出错的原因可能有很多。到底是什么原因？你完全没经验只能瞎排查，还不一定能排查成功，到头来白折腾一场。这太伤神了好吗？！都是没人自己指点自己瞎折腾的恶果啊。不过经过那么久的尝试，我已经对失败很有耐心了。

三、在GitHub上创建项目主页

第二步我把代码推送到了我在GitHub上的仓库中，但是它们是在master分支上被提交的，因此还不能通过http:// user-id.github.io/project-name来访问我第一步做好的主页。这是我需要用GitHub上提供的pages服务来为自己的项目创建项目主页。

为项目启用项目主页需要在项目版本库中创建一个名为gh-pages的分支，并向其中添加静态网页。
第二步之后，我的try2code版本库只有一个名为master的分支，如果直接从master分支创建gh-pages分支操作非常简单，但是作为保存网页的gh-pages分支中的内容和master分支中的可能完全不同。如果不希望gh-pages分支继承master分支的历史和文件，即想要创建一个干净的gh-pages分支，需要一点技巧。
可以依次执行以下命令行来创建干净的gh-pages分支：

$ git checkout -b gh-pages   #基于master分支建立gh-pages分支
$ rm .git/index              #清空暂存区
$ printf "hello world.\n" > index.html  #创建项目主页index.html，之后把它改成你自己写的index.html即可
$ git add index.html  #跟踪index.html文件，把它添加到暂存区。
$ git reset --hard $(echo "branch gh-pages init." | git commit-tree $(git write-tree)) #用Git底层命令创建新的根提交，并将分支gh-pages重置
$ git push -u origin gh-pages  #创建新的根提交，并将分支gh-pages重置

做了以上工作后，就可以通过http:// user-id.github.io/project-name 即http://yxuey.github.io/try2code来访问项目主页了。

成功完成了以上事情真的很有成就感，以后也会努力获取这种成就感的！虽然才笨拙地开始起步，虽然我一直走得比别人慢，虽然我不确定在前端这条路上我能走到哪一步，但我还是想战胜自己的疑虑并继续尝试。将来因坚持不下而放弃时，希望我还能安慰自己：“你的挣扎，虽败犹荣”
