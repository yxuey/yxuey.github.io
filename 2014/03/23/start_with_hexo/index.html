开始使用独立博客

By YuXueying

3月 23 2014 更新日期:5月 5 2014

文章目录

欢迎来到我的个人博客！这是我第一次尝试搭建个人博客，折腾了两个晚上加一个下午才初步搭建完善。如果你也对搭建个人博客有兴趣，不妨参考这两个教程：hexo搭建博客系列教程、使用hexo搭建静态博客。

我为什么会忽然想搭建个人博客呢？前几天在网上找实习时看到某些意向岗位的要求是 “精通什么什么熟悉什么什么，在GitHub上有技术沉淀的优先考虑。” 当时就去查了一点它的资料，知道了它是全球最大的社交编程及代码托管网站，可以用来部署个人博客等等。没过几天，又在QQ上看到一个学长用hexo在GitHub上搭建好了博客，而且据他说hexo搭建个人博客相对比较简单。于是就动了心思。

“hexo是由台湾的tommy351创建的简易强大的博客框架。具有静态页面生成速度快的的好处，尤其是当博文数量较多时，这种优势会更加明显。”

好吧，我这种小白就是看上了简易快速这一好处。于是开始跟着教程折腾了，折腾过程中还是出了一些岔子的，重装软件、配置参数屡屡失败也几度让我想砸电脑。最后还是折腾出一些经验来了，还好没有放弃。所以如果看到这篇博文的你也想用hexo搭博客的话，也许我可以帮上你一些小忙。

前阵子一直在犹疑该选择产品方向还是前端开发方向，说到底其实是因为自己太弱太焦躁太怕承当不起选错的后果。产品方向需要你眼界开阔、对互联网产品要有敏感度、要有想法有领导力有协调能力，听着就觉得自己明显不太符合要求。前端开发呢，呵呵，我害怕hold不住代码我害怕折腾半天还是会失败。照说我应该见到代码就绕道，但我还是觉得前端开发很有趣，经过一番努力然后做出成果的成就感也很美妙。也有可能是虚妄的自我认同感在作祟吧，我就是不死心，还是想继续尝试学习前端、想一开始工作时从事一些跟技术比较相关的工作。

于是也把搭个人博客当成了一件辅助我做决定的事情。我当时想“十几天都搭不好的话就彻底死了前端开发的心吧，连这点事都搞不定你必然hold 不住前端的。” 如果搭建成功了，那可以试着继续走下去，我也会更有动力学更多东西然后把学习笔记搬到这个博客上。毕竟如果只是写写心情吐吐槽大可在简书微博空间上解决，既然大费周章的建了博客就要努力让博客里多一些技术类文章，不然真是太丢人了。另外，有了这个博客后，我也许会更有动力学前端的知识来优化自己的博客。 (我要像。。。像妈妈打理自己的孩儿们一样打理它！！) 这个博客现在无论是界面还是内容都略显笨拙粗糙呀。以后也可能会买个域名、设置一下DNS啥的，为了追求访问速度也有可能会把它部署到国内的平台上。学了移动Web优化技巧后也会尝试对个人站点做些改进，以追求更好的移动端访问效果。

总之，这里将主要做为我的学习笔记的集放地，这样既不必担心资料丢失也可方便共享交流。也可能偶尔会在这里写些杂七杂八的生活记录等等，有些事情还是值得细心用文字记下来的。
其实本篇文章的发表主要做测试之用，若顺利，以后将陆续把笔记、教程贴到这里。
